# Contributing to BotTimer

Thank you for your interest in contributing! BotTimer is an individual project, so please keep in mind that responses to issues and pull requests may be slower than in larger projects.

## Bug Reports

When filing a bug report, please include:

- **Device**: iPhone / iPad / Android device model name
- **OS version**: e.g. iOS 18.3 or Android 14
- **App version**: shown in Settings screen
- **Steps to reproduce**: numbered step-by-step
- **Expected behavior** vs **actual behavior**
- Screenshots or screen recordings if helpful

## Feature Requests

Open a GitHub Issue with a clear description of the feature and why it would be useful.  
There is no guarantee that all requests will be implemented, but all feedback is appreciated.

## Pull Requests

Before submitting a pull request, please **open an Issue first** to discuss the change.  
This avoids wasted effort if the direction doesn't align with the project's goals.

When submitting a PR:

1. Fork the repository and create a branch from `master`
2. Follow the existing code style (TypeScript + React Native)
3. Test on a real device if possible
4. Keep the scope of the change focused
